Protocolos — las reglas del juego

Cómo dos computadoras logran "hablar el mismo idioma".

📖 Concepto

Cuando dos personas se saludan en Argentina siguen un protocolo sin pensarlo: extender mano derecha, apretar fuerte, sacudir 2-3 veces, soltar. Si alguien rompe esa secuencia (mano izquierda, no suelta, sacude 20 veces), la situación se vuelve incómoda — el "protocolo" se rompió.

Un protocolo es exactamente eso: un conjunto de reglas que dos partes acuerdan seguir para comunicarse. Sin protocolo, no hay comunicación posible.

Los 3 pilares de cualquier protocolo

📐 Sintaxis — la forma

Cómo se ve el mensaje: estructura, campos, orden de bytes. El sobre tiene remitente arriba a la izquierda y sello arriba a la derecha.

💡 Semántica — el significado

Qué significa cada parte. El campo "sucursal" en un formulario significa algo específico — escribir tu nombre ahí rompe la semántica.

⏱️ Temporización — el cuándo

Quién habla primero, cuánto se espera, qué pasa si no llega respuesta. En walkie-talkie decís "cambio" para indicar que terminaste.

Asincrónico vs Sincrónico

  • Asincrónico (WhatsApp): cada mensaje tiene marca de inicio y fin. No hay un ritmo común. Cada carácter viaja con un bit de arranque y bit de parada.
  • Sincrónico (videollamada): ambos están conectados al mismo tiempo, mismo "reloj" compartido. Más eficiente, requiere sintonía.

Características clave

  • Control de errores: info extra para detectar si algo se corrompió.
  • Control de flujo: el receptor puede decir "pará un poco" si llega rápido.
  • Control de secuencia: los mensajes se numeran para reordenar si llegan desordenados.
  • Fragmentación: los datos grandes se cortan en trozos para viajar.
  • Transparencia: si los datos contienen un patrón que parece señal de control, se marca para no confundirse.

Un protocolo no es magia — es un contrato. "Yo te mando los datos en este formato, vos respondés de esta manera". HTTP, TCP, IP, WiFi son protocolos: reglas documentadas que todos acordaron seguir.
