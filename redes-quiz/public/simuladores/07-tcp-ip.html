TCP/IP — handshake y puertos

El 3-way handshake, comparativa TCP vs UDP, y para qué sirven los puertos.

📖 Concepto

TCP/IP es el modelo real que usa Internet. Tiene 4-5 capas (Aplicación, Transporte, Internet, Acceso a la red, Física). Lo más importante de este capítulo: la diferencia entre TCP y UDP, y qué son los puertos.

TCP vs UDP

📬 TCP — Correo certificado

Mandás la carta. El cartero te trae el acuse firmado. Si no llega en X tiempo, va de nuevo.

  • Datos completos ✓
  • En orden ✓
  • Sin duplicados ✓
  • Más lento (espera ACKs)

Usar para: web, descargas, email — todo donde no puede faltar nada.

📨 UDP — Folleto en el buzón

Lo tirás y listo. No sabés si lo leyeron. Si se pierde, no te enterás.

  • Sin garantías
  • Sin orden garantizado
  • Header mínimo
  • Muy rápido

Usar para: videollamada, streaming, juegos, DNS — donde la velocidad importa más que la perfección.

3-way handshake (solo TCP)

Antes de mandar datos, TCP negocia la conexión con tres mensajes:

  • SYN — el cliente le dice al servidor "quiero conectarme, mi número de secuencia inicial es X"
  • SYN-ACK — el servidor responde "OK, mi número es Y, confirmo el tuyo (X+1)"
  • ACK — el cliente confirma "OK, recibí tu Y (Y+1)"

Recién después se mandan los datos reales. UDP no tiene handshake — manda directo.

Puertos: las puertas del edificio

La IP identifica la máquina (el edificio). El puerto identifica la app (el departamento). Tu PC corre Chrome y Spotify a la vez con la misma IP — los puertos los distinguen.

  • Bien conocidos (0–1023): 22 SSH · 25 SMTP · 53 DNS · 80 HTTP · 443 HTTPS
  • Dinámicos (1024–65535): los asigna el SO a tus apps cuando inician una conexión

Encapsulación — las muñecas rusas

Cada capa envuelve los datos: [ETH | IP | TCP | datos | CRC]. Al recibir, cada capa quita su envoltura y pasa el resto a la de arriba.
