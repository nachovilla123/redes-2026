Fragmentación IPv4

Calculá ID, MF y Fragment Offset cuando un paquete excede el MTU del próximo enlace.

📖 Concepto

Cada tecnología de red tiene un MTU (Maximum Transmission Unit) — el máximo de bytes que puede transportar en una sola trama. Si un paquete IP es más grande que el MTU del próximo enlace, hay dos opciones: descartarlo, o fragmentarlo.

MTU típicos

TecnologíaMTU
Ethernet1500 bytes
WiFi (802.11)2304 bytes
PPP (WAN)576 - 1500 bytes

¿Cuándo se fragmenta?

Cuando el paquete > MTU del próximo enlace Y el bit DF (Don't Fragment) está en 0.

Si DF=1 y el paquete no entra → router descarta + ICMP "Fragmentation Needed" al origen.

Campos del header IPv4 para fragmentación

  • Identification (16 bits): identifica a qué datagrama pertenece. Todos los fragmentos del mismo datagrama tienen el mismo ID.
  • DF (1 bit): 1 = no fragmentar, 0 = se puede.
  • MF — More Fragments (1 bit): 1 = hay más fragmentos, 0 = este es el último.
  • Fragment Offset (13 bits): posición del fragmento dentro del datagrama original, en unidades de 8 bytes.

Cómo calcular

Datos por fragmento = MTU − IP_header, redondeado hacia abajo al múltiplo de 8 (porque el offset se cuenta en bloques de 8 bytes).

¿Quién reensambla?

Solo el destino final. Los routers intermedios no reensamblan — solo fragmentan más si hace falta. Si un fragmento se pierde, se descarta el datagrama completo y TCP (o la app) tiene que reenviar.

Tip: IP fragmenta sin garantizar entrega. Las garantías las da TCP encima.
