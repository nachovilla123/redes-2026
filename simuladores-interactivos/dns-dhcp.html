Controles

Proceso DORA
Discover → Offer → Request → Ack. Tu PC recién conectada no tiene IP — pide una al servidor DHCP.
Estado actual
PC se conectó a la red sin IP. Apretá "Siguiente paso".
Datos que entrega
IP · Máscara · Gateway · DNS · Lease time
Puertos: UDP 67 servidor / UDP 68 cliente.
¿Por qué broadcast?
El cliente no conoce la IP del servidor DHCP — y ni tiene IP propia. Broadcast es el único mecanismo. El paso REQUEST también es broadcast para que los demás servidores DHCP liberen su oferta.
Resolución iterativa
El resolver pregunta a cada servidor, que responde "yo no sé pero preguntale a tal". Va escalando hasta el autoritativo.
Paso actual
Vas a resolver www.google.com. Apretá "Siguiente paso".
Tipos de servidores DNS
Stub resolver: tu SO. Solo pregunta.
Recursive resolver: hace todo el trabajo (8.8.8.8, 1.1.1.1).
Root server: sabe dónde está cada TLD.
TLD server: sabe los NS de cada dominio bajo .com, .ar, etc.
Authoritative: tiene la respuesta final.
Caché y TTL
Cada respuesta tiene un TTL en segundos. El resolver guarda en caché hasta que expira → la siguiente consulta es instantánea sin volver a recorrer toda la jerarquía.
